1. Introduction

WrapServices is a production-ready, multi-role service marketplace platform built with React 18, Vite, TypeScript, Tailwind CSS, and a managed Supabase (Lovable Cloud) backend. It supports four user roles — Customer, Service Provider, Employer, and Admin — with end-to-end flows for bookings, job posting, applications, payments, reviews, messaging, and verification.

  • Fully responsive PWA with installable mobile experience
  • Role-based dashboards and protected routes
  • Realtime messaging, notifications, and live updates
  • Provider subscriptions, sponsored services, and pay-per-job posting
  • Admin suite: users, verifications, payments, jobs, analytics, CMS, banners, coupons

2. Installation Guide

  1. Clone the repository or remix the Lovable project.
  2. Install dependencies: npm install (or bun install).
  3. Copy .env.example to .env and fill in Supabase keys (see §3).
  4. Run the dev server: npm run dev.
  5. Open http://localhost:8080.

Requires Node 18+ and a Supabase project (or use Lovable Cloud — included by default).

3. Environment Setup (.env)

Three public variables are required at build time:

VITE_SUPABASE_URL="https://<project-ref>.supabase.co"
VITE_SUPABASE_PUBLISHABLE_KEY="<anon key>"
VITE_SUPABASE_PROJECT_ID="<project-ref>"

Server-side secrets (edge functions only) — set via the Lovable secrets manager:

  • SUPABASE_SERVICE_ROLE_KEY — admin operations
  • LOVABLE_API_KEY — AI gateway (smart matching, descriptions)
  • DEMO_RESET_SECRET — gated demo data reset endpoint

4. Supabase Integration Guide

The client is pre-wired at src/integrations/supabase/client.ts. Schema is managed through versioned migrations under supabase/migrations/.

Core tables (57 total)

  • Identity: profiles, user_roles, provider_staff, employer_profiles
  • Marketplace: vendor_services, tasks, task_proposals, bookings, reviews
  • Money: wallets, wallet_transactions, payment_transactions, invoices, withdrawals, refund_requests, category_commissions
  • Engagement: conversations, messages, notifications, favorites, saved_jobs
  • CMS / Admin: cms_pages, homepage_content, banners, advertisements, platform_settings, promo_coupons, email_templates
  • Trust & Safety: vendor_verifications, disputes, content_reports, admin_audit_log

Storage buckets

  • Public: avatars, branding, tenant-logos, portfolio, task-photos, review-photos, chat-attachments
  • Private: verification-docs, receipts, demo-gallery

Edge functions

Deployed: seed-demo-users, reset-demo-data, send-booking-email, generate-booking-receipt, generate-task-description, smart-match-vendor, invite-staff, resend-staff-invitation, validate-coupon, geocode-search, review-reminder, send-push, dispatch-notification-channels.

5. Authentication System

Email/password authentication powered by Supabase Auth with optional Google OAuth. Sessions persist via localStorage with auto-refresh.

  • Sign up → handle_new_user() trigger creates a profile + assigns role from user_metadata.role.
  • Sign in → log_login_event() writes to admin_audit_log.
  • Soft-delete via profiles.deleted_at blocks sign-in and signs the user out.
  • Profile completion gate (/complete-profile) for first-time users.

6. Role-Based Access System

Roles live in a separate user_roles table (never on profiles) and are checked via the has_role(uuid, app_role) security-definer function.

RoleDashboardCapabilities
client (customer)/client-dashboardBrowse services, book, post tasks, leave reviews
provider/provider-dashboardList services, apply to jobs, manage bookings, sponsor listings
provider_staff/staff-dashboardOperate on behalf of a provider (treated as provider for UI gating)
employer/employer-dashboardPost jobs, review applicants, hire providers
admin/adminFull platform access — users, payments, moderation, settings

Frontend gates: <ProtectedRoute requiredRole>, <Can>, and the usePermission() hook.

7. PWA System

  • Service worker via vite-plugin-pwa with Workbox precaching and background sync.
  • Push notifications through public/push-sw.js + send-push edge function.
  • Per-tenant branding (light/dark/PWA logo, theme color, manifest) injected at runtime from platform_settings via usePwaBranding().
  • Install prompt managed by usePwaInstall() with iOS guidance fallback.

8. Setup Wizard System

New users complete onboarding via OnboardingWizard.tsx:

  1. Choose role and basic identity
  2. Complete profile (display name, phone, address, bio, avatar)
  3. Role-specific extras (business name + categories for providers, company for employers)
  4. Optional: take the interactive product tour

Admins configure platform-wide defaults at /admin/platform-settings (branding, logos, theme, email).

9. Navigation System (Responsive)

  • Desktop (≥1024px): UnifiedHeader + sticky ProfileRail sidebar on dashboards.
  • Tablet (768–1023px): Collapsible sidebar with icon-only mode.
  • Mobile (<768px): Hamburger drawer; bottom nav pills removed in v2.1 for a cleaner surface.
  • Header z-index: 60 (above portaled dropdowns at 50) to keep sticky header visible.
  • Container width: main content capped at max-w-5xl for readability.

10. Demo / Test Mode System

Demo accounts are seeded via the seed-demo-users edge function. Credentials are listed on /demo-login.

  • Admin, Provider, Customer, Employer, and Staff demo logins
  • Mock services, bookings, reviews, conversations, and notifications
  • Auto-reset every 6 hours via reset-demo-data (gated by DEMO_RESET_SECRET)

11. Deployment Guide

  1. Click Publish in Lovable (top-right) to deploy the frontend to *.lovable.app.
  2. Backend changes (migrations, edge functions) deploy automatically when approved in chat.
  3. Connect a custom domain under Project Settings → Domains.
  4. Run npm run build to produce a static dist/ if self-hosting.
  5. Self-host: any static host (Vercel, Netlify, Cloudflare Pages) — set the three VITE_* env vars.

12. Troubleshooting

"Permission denied for table" from PostgREST
The table is missing a GRANT. Every public-schema table needs explicit grants to authenticated / service_role.
Realtime channel never fires
Register all .on() handlers before .subscribe(), and use a unique channel id.
Sticky header hides when a dropdown opens
Fixed in v2.1.0 — Radix scroll-lock is neutralized in index.css and header is z-[60].
"Unsupported provider" on Google login
Enable Google provider in the Lovable Cloud auth settings.

13. FAQ

Can I rebrand the platform?
Yes — every color, logo, and copy block is editable. Per-tenant branding overrides primary/accent at runtime.
Does it support multi-tenancy?
Path-based multi-tenancy is included with RLS-isolated data per tenant.
What payment providers are supported?
Stripe and PayPal scaffolding with 25% deposit logic and commission tracking.
Is the source code editable?
Yes — full TypeScript source ships in the repo.
